# Security Policy

## Supported versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security bugs.

Email security reports to the maintainers (use the contact address listed on the GitHub org or your team's security inbox). Include:

- A description of the issue and impact
- Steps to reproduce
- Affected versions
- Any suggested fix, if you have one

We aim to acknowledge reports within **7 days** and provide a remediation timeline when possible.

## Scope

In scope: the `llm_obs` Python package (PII redaction, transport, provider wrappers, and auto-instrumentation).

Out of scope: hosted ingestion backends, dashboards, or deployments you operate separately — report those to the respective service owners.

## HackerOne

If this project runs a public disclosure program on [HackerOne](https://www.hackerone.com/product/community-edition), the program URL will be linked here once it is live.
